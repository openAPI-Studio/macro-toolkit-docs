Installation

VS Code Extension

Search "API Studio" in the Extensions marketplace, or install via the command palette:

ext install open-post.open-post

Requires VS Code 1.85.0 or later. Open with Ctrl+Alt+P / Cmd+Alt+P.

CLI (npm)

# Global install (recommended)
npm install -g openpost
openpost --help

# Or run without installing
npx openpost --help

Storage Layout

All data lives on your machine as plain JSON — nothing is sent to the cloud.

.openpost/                 ← workspace-local (per project)
  ├── collections.json
  ├── environments.json
  ├── history.json
  ├── snapshots.json
  ├── cookies.json
  ├── session.json
  └── mock-servers.json

~/.openpost/global/        ← shared across all workspaces
  ├── collections.json
  ├── environments.json
  ├── config.json
  └── vault.enc            ← AES-256-GCM encrypted secrets

Protocol Support

REST / HTTP

Arbitrary HTTP methods with URL template variables, all body types, custom headers and params. Protocol Selector button switches between HTTP, GraphQL, WebSocket, and gRPC.

  • Body types: JSON, raw, XML, form-data, urlencoded, binary, GraphQL
  • CodeMirror 6 editor with syntax highlighting, Ctrl+F search, zoom controls
  • Inline comment stripping, {{variable}} color tokens
  • cURL paste import with full flag support
GraphQL

Full Postman/Insomnia parity. Schema-aware editing, visual query builder, docs panel, and live subscriptions.

  • Schema introspection, SDL file import, schema URL override, persistent cache
  • Visual query builder — checkbox field tree with auto-generated variable declarations
  • Docs panel with full-text search, clickable type refs, breadcrumb navigation
  • Subscriptions via graphql-transport-ws — live event panel with timestamps
  • Multi-operation picker, inline lint squiggles, prettify button
WebSocket

Activated when the URL starts with ws:// or wss://.

  • Message composer with Text / JSON type selector
  • Read-only message log with direction filter (All / Sent / Received)
  • Auth and custom headers injected into the WS handshake
gRPC

Activated with grpc:// / grpcs:// URLs.

  • Proto file picker, service and method dropdowns
  • Unary and server-streaming call types
  • Metadata headers tab, streaming message log with timestamps, grpcurl export

Collections & Environments

Collections

  • Local (workspace) and global (user) scope
  • Unlimited nested folders with drag-and-drop reorder and reparent
  • Deep search by request name or URL across nested folders
  • Postman v2.0 / v2.1 import, OpenAPI & Swagger import
  • Collection-level variables, auth, and scripts cascade to all requests
  • Collection runs — sequential or parallel (configurable concurrency)

Environments & Variables

  • Local and global environments with active selection
  • {{variable}} interpolation in URL, headers, body, and auth
  • Response variable extraction (JSON path, header, regex, full body)
  • Priority: collection vars < global env < vault secrets < local env
  • Enabled/disabled rows, visual distinction for active global env

Testing & Scripting

GUI Test Rules (no-code)

11 rule types in the Tests tab — results saved with each history entry.

status time size header body body contains body is JSON body schema JSON path content type content length

JavaScript Scripts

Sandboxed VM context with a 5-second timeout. No filesystem or network access.

  • Pre-request script runs before HTTP execution
  • Test script runs after response is available
  • Sandbox exposes request, response, environment, console
  • Scripts inherit from collection → folder → request

Script Examples

Pre-request — add a dynamic header

request.headers.push({
  key: 'X-Timestamp',
  value: new Date().toISOString(),
  enabled: true
});

Test script — assert JSON response

const data = response.json();
console.assert(Array.isArray(data.items), 'items must be an array');
console.assert(response.status === 200, 'expected 200, got ' + response.status);

Execution Order

  1. 1Auth headers injected
  2. 2Pre-request script runs
  3. 3Cookies attached
  4. 4HTTP request executes
  5. 5Response cookies parsed and persisted
  6. 6GUI variable extraction rules run
  7. 7GUI test rules evaluated
  8. 8Test script runs

Mock & MCP Servers

Mock Servers

  • Multiple server instances with auto-start
  • Route-based matching with custom bodies or snapshot responses
  • File-served responses, per-server request logs
  • {{variable}} interpolation in response bodies and headers
  • Optional Swagger/OpenAPI exposure per server
  • Fully offline — no cloud infrastructure

MCP Server Tools AI

Unique to API Studio — expose your API collections as MCP tools for AI coding assistants.

  • Collection-sourced MCP server — one per collection
  • Per-request MCP description & JSON Schema (auto-generate)
  • Works with Claude Desktop, Cursor, VS Code Copilot
  • Management MCP server — 32 workspace tools on port 3199
  • Mock routes can also be exposed as MCP tools
  • Headless via CLI: openpost serve

Connecting an AI Client

# Start the management MCP server
openpost serve

# Output:
#   Listening on http://127.0.0.1:3199/mcp
#   32 tools available

# Claude Desktop config (~/.config/claude/claude_desktop_config.json)
{
  "mcpServers": {
    "api-studio": { "url": "http://127.0.0.1:3199/mcp" }
  }
}

Secret Vault

A built-in AES-256-GCM encrypted vault for API keys, passwords, and TLS certificates. Stored in ~/.openpost/global/vault.enc — never leaves your machine.

Vault Features

  • AES-256-GCM encryption with PBKDF2 key derivation
  • Master password create, unlock, lock, and change flows
  • Secret sets — map vault keys to {{variable}} names
  • TLS certificates: PEM, PFX/DER, CA bundles with optional passphrase
  • Vault secrets scrubbed from all persisted history entries
  • Interpolation priority: vault overrides global env

CLI Vault Commands

# Create vault
openpost vault create

# Unlock
openpost vault unlock

# Add a secret
openpost vault add-secret api_key sk-live-abc

# List (masked)
openpost vault list-secrets
# api_key  ••••••••
# db_pass  ••••••••

# View actual values
openpost vault list-secrets --json

CLI Reference

The openpost CLI reads the same .openpost/ files as the VS Code extension — no separate setup required.

Run a Collection

openpost run "My API" --env production
openpost run "My API" --env staging --mode parallel --concurrency 10
openpost run "My API" --filter "GET*" --verbose
openpost run "My API" --output-file results.json

Exit Codes

1Failure — one or more requests or tests failed
2Error — bad arguments, resource not found, config error

CI/CD Integration

Use openpost run in any CI/CD pipeline. Exit code 1 when tests fail, 0 on success.

GitHub Actions

- name: API Tests
  run: |
    npm install -g openpost
    openpost run "My API" --env staging --quiet
    
# Save results as artifact
- name: API Tests with report
  run: |
    openpost run "My API" --env staging --output-file results.json --json
- uses: actions/upload-artifact@v4
  with:
    name: api-test-results
    path: results.json

Programmatic Usage (Node.js)

import { OpenPost } from 'openpost';

const op = new OpenPost({ workspace: '/path/to/project' });

// Run a full collection
const entry = await op.runCollection('My API', {
  env: 'production',
  mode: 'sequential',
  onProgress: (r) => console.log(`${r.passed ? '✓' : '✗'} ${r.method} ${r.url}`),
});
console.log(`${entry.passed} passed, ${entry.failed} failed`);

// Run a single request
const { response, testResults } = await op.runRequest('My API', 'Get Users', {
  env: 'prod'
});

await op.dispose();
